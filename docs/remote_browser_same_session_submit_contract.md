# Remote Browser Same-Session Flow Submit Contract

## Endpoint

`POST /api/v1/sessions/{session_id}/flow-request`

## Request Body

```json
{
  "method": "POST",
  "url": "https://aisandbox-pa.googleapis.com/v1/video:batchAsyncGenerateVideoText",
  "headers": {
    "authorization": "Bearer ...",
    "content-type": "application/json",
    "x-browser-channel": "stable",
    "x-browser-copyright": "Copyright 2026 Google LLC. All Rights reserved.",
    "x-browser-validation": "...",
    "x-browser-year": "2026"
  },
  "json": {},
  "timeout": 60,
  "expect_json": true
}
```

## Behavior Requirements

The remote browser service must:

- Use the **same browser/page/session** that solved reCAPTCHA.
- Use the **same network identity / proxy** bound to that browser session.
- Execute final Flow fetch **inside** that browser context.
- Let browser-native identity headers be generated by browser runtime (UA / client hints / fetch metadata).
- Return:
  - `status_code`
  - `headers`
  - `body`
  - parsed `json` (when available)
  - `duration_ms`
  - `same_session: true`

## Why

This avoids trust mismatch:

1. solve in one identity
2. submit in another identity

Same-session solve + submit keeps reCAPTCHA and final Flow submission aligned.
